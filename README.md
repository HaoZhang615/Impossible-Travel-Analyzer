# Impossible Travel Analyzer

A multi-agent investigation workflow for **Impossible Travel** detections using the **Microsoft Agent Framework** and **WorkflowBuilder** graph-based orchestration.

## Architecture

```
                                  ┌──> [LocationAnomaly]
                                  ├──> [AuthenticationAnomaly]
                                  ├──> [TokenAnomaly]
                                  ├──> [BruteForcePattern]
                                  ├──> [MfaAbuse]
Sentinel Detection ──> Context ──┤──> [PostLoginActivity]
                       Enricher  ├──> [OAuthAbuse]
                                  ├──> [MailboxManipulation]
                                  ├──> [HighPrivilegeUser]
                                  └──> [DisabledAccountSignIn]
                                              ↓ (all 10)
                                       fan-in → Aggregator → PACO
```

- **Context Enricher** — deterministic `Executor` (no LLM), queries AD/Entra user context
- **10 Risk Sub-Agents** — each evaluates one risk factor in parallel via fan-out, produces structured `RiskEvidence`
- **Risk Aggregator** — deterministic `Executor` (no LLM), assembles `ThreatDecisionRecord`
- **PACO** — LLM-powered decision maker, produces final verdict with action plan

## Prerequisites

- Python 3.10+
- [Azure CLI](https://learn.microsoft.com/cli/azure/install-azure-cli) (`az login` completed)
- A [Microsoft Foundry project](https://learn.microsoft.com/azure/foundry/how-to/create-projects) with a deployed model (e.g., `gpt-4o`)
- Docker Desktop (for Hosted Agent deployment only)

## Quick Start

```bash
pip install -r requirements.txt
```

Open `00-setup.ipynb` and edit the Foundry connection variables, then run each notebook in sequence.

## Features

| Feature | Description |
|---|---|
| Fan-out/fan-in orchestration | 10 risk agents execute in parallel via `WorkflowBuilder` |
| Structured output | All agents produce typed Pydantic models |
| DevUI visualization | Interactive graph of the 13-node investigation pipeline |
| Hosted Agent deployment | Deploy as a container to Foundry Agent Service |
| Simulated data | Hardcoded sign-in logs, audit logs, IP reputation for demo |

## Notebooks

| Notebook | Description |
|---|---|
| `00-setup.ipynb` | Install dependencies, configure Foundry connection, verify connectivity |
| `01-investigation.ipynb` | Define models, build agents/workflow, run investigation, view results, launch DevUI |
| `02-deploy.ipynb` | Generate deployment artifacts, build/push Docker image, deploy as Hosted Agent |
